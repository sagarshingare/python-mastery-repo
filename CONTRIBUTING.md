# Contributing to Python Mastery Repository

Thank you for contributing! This repository is designed to be a community-first learning resource with strong engineering practices.

## Contribution process

1. Fork the repository.
2. Create a feature branch from `main`.
3. Add code with tests and documentation.
4. Run `make test` locally.
5. Open a pull request with a clear summary and related issue.

## Coding standards

- Follow PEP 8
- Use type hints
- Write modular, reusable functions
- Add docstrings and comments for behavior and intent
- Include tests for edge cases
- Avoid hard-coded credentials and secrets

## Documentation

- Add or update README files for new modules
- Include diagrams and architecture notes when relevant
- Document assumptions, tradeoffs, and performance implications
